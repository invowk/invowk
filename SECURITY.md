# Security Policy

## Reporting a Vulnerability

**Do not report security vulnerabilities through public issues.**

Please use [GitHub Security Advisories](https://github.com/invowk/invowk/security/advisories/new) to report vulnerabilities privately.

When reporting, please include:

- Type of vulnerability (e.g., command injection, path traversal, privilege escalation)
- Affected files or components, if known
- Steps to reproduce the issue
- Potential impact or severity

## Response Timeline

- **Initial response**: within 48 hours
- **Status update**: within 7 days

## Coordinated Disclosure

We follow a coordinated disclosure process. After a fix is available, we will work with you on an appropriate disclosure timeline. We ask that you do not publicly disclose the vulnerability until a fix has been released.
