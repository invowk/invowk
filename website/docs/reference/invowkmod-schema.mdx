# Invowkmod Schema Reference

:::warning Alpha — Schema May Change
The invowkmod schema is still evolving. Fields, types, and validation rules **may change between releases** as we stabilize the format. Always check the [changelog](https://github.com/invowk/invowk/releases) when upgrading.
:::

Complete reference for `invowkmod.cue`, the module metadata file. Module metadata is validated as a closed struct, so unknown fields cause a validation error.

## Root Structure

Every module must include an `invowkmod.cue` file with this structure:

<Snippet id="reference/invowkmod/root-structure" />

### module

**Type:** `string` (pattern: `^[a-zA-Z][a-zA-Z0-9]*(\.[a-zA-Z][a-zA-Z0-9]*)*$`)
**Required:** Yes

The module identifier. Local installed module directories use this value as `<module-id>.invowkmod`. For explicit dependency subpaths, the selected directory basename must match this prefix before `.invowkmod`. For a root Git repository source, the repository basename does not have to match.

<Snippet id="reference/invowkmod/module-examples" />

### version

**Type:** `string` (pattern: `^(0|[1-9][0-9]*)\.(0|[1-9][0-9]*)\.(0|[1-9][0-9]*)(-[0-9a-zA-Z-]+(\.[0-9a-zA-Z-]+)*)?$`)
**Required:** Yes

Module version in semver format (e.g., `"1.0.0"`, `"2.1.0-beta.1"`).

### description

**Type:** `string`
**Required:** No

A short summary of the module's purpose.

### author

**Type:** `string`
**Required:** No

Module author or maintainer (free-form text, typically "Name \<email\>" or organization name).

### license

**Type:** `string`
**Required:** No

Module license using an SPDX identifier (e.g., `"MIT"`, `"Apache-2.0"`, `"MPL-2.0"`).

### repository

**Type:** `string` (pattern: `^(https://|git@|ssh://)`)
**Required:** No

Canonical source URL for this module. Only `https://`, `git@`, and `ssh://` schemes are accepted.

### requires

**Type:** `[...#ModuleRequirement]`
**Required:** No

List of module dependencies from Git repositories.

## ModuleRequirement

Defines a single dependency entry:

<Snippet id="reference/invowkmod/requirement-structure" />

### git_url

**Type:** `string`
**Required:** Yes

Git repository URL. Accepted forms: HTTPS (`https://...`), SSH shorthand (`git@...`), or SSH URI (`ssh://...`). The repository basename does not have to end in `.invowkmod`.

When `path` is omitted, the fetched repository root must contain `invowkmod.cue` and `invowkfile.cue`. Child modules are not auto-selected; declare `path` for monorepos.

### version

**Type:** `string`
**Required:** Yes

Semantic version constraint used to resolve a Git tag. Supported forms are a bare semantic version or one operator from `^`, `~`, `>`, `>=`, `<`, `<=`, or `=` followed by a semantic version. The constraint value must not use a `v` prefix, trailing junk, or compound ranges such as `>=1.0.0 <2.0.0`.

### alias

**Type:** `string`
**Required:** No

Overrides the default command source ID for this dependency. Aliases use command source ID rules: they must start with a letter and may contain letters, digits, `.`, `_`, or `-`.

### path

**Type:** `string`
**Required:** No

Subdirectory within the repository that contains the module. Must be relative, cannot contain path traversal sequences, and must point directly to a directory whose basename ends in `.invowkmod`. The basename prefix must match the selected module's `module` value.
