# Invkpack Schema Reference

:::warning Alpha — Schema May Change
The invkpack schema is still evolving. Fields, types, and validation rules **may change between releases** as we stabilize the format. Always check the [changelog](https://github.com/invowk/invowk/releases) when upgrading.
:::

Complete reference for `invkpack.cue`, the pack metadata file. Pack metadata is validated as a closed struct, so unknown fields cause a validation error.

## Root Structure

Every pack must include an `invkpack.cue` file with this structure:

<Snippet id="reference/invkpack/root-structure" />

### pack

**Type:** `string` (pattern: `^[a-zA-Z][a-zA-Z0-9]*(\.[a-zA-Z][a-zA-Z0-9]*)*$`)  
**Required:** Yes

The pack identifier. It must match the folder name prefix (before `.invkpack`).

<Snippet id="reference/invkpack/pack-examples" />

### version

**Type:** `string` (pattern: `^[0-9]+\.[0-9]+$`)  
**Required:** No

Pack metadata version. Current version is `"1.0"`.

### description

**Type:** `string`  
**Required:** No

A short summary of the pack’s purpose.

### requires

**Type:** `[...#PackRequirement]`  
**Required:** No

List of pack dependencies from Git repositories.

## PackRequirement

Defines a single dependency entry:

<Snippet id="reference/invkpack/requirement-structure" />

### git_url

**Type:** `string`  
**Required:** Yes

Git repository URL (HTTPS or SSH). The repository must contain a pack.

### version

**Type:** `string`  
**Required:** Yes

Semantic version constraint used to resolve a Git tag.

### alias

**Type:** `string`  
**Required:** No

Overrides the default namespace for this dependency (must follow pack naming rules).

### path

**Type:** `string`  
**Required:** No

Subdirectory within the repository that contains the pack. Must be relative and cannot contain path traversal sequences.
