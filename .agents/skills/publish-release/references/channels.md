# Release Channels

Use this reference to verify what a release publishes and which channels should change.

## Primary Pipeline

- `make release` and `make release-bump` call `scripts/release.sh`.
- Real maintainer releases require a clean `main`, an unused `vMAJOR.MINOR.PATCH[-label.N]` tag, a markdown `RELEASE_NOTES_FILE`, and reachable `origin`.
- Tags are signed annotated tags. The tag message carries the reviewed release notes so CI can recover the body after checkout.
- `.github/workflows/release.yml` runs tests first, extracts release notes from the tag, tracks release performance in Bencher, then runs GoReleaser.
- `workflow_dispatch` can create an annotated tag inside GitHub Actions. It validates `main`, requires markdown notes input, supports dry run, and uses an unsigned bot-created annotated tag.
- CI also exercises release health before tag day through GoReleaser checks/dry-run coverage. If release config changed, inspect `.github/workflows/ci.yml` in addition to the dedicated release workflow.

## GitHub Release And Artifacts

GoReleaser publishes:

- GitHub Release in `invowk/invowk`.
- Linux, macOS, and Windows archives.
- `.tar.gz` archives for Linux/macOS and `.zip` archives for Windows.
- `linux` and `darwin` for `amd64` and `arm64`; `windows` for `amd64`. Windows `arm64` is intentionally ignored.
- `amd64` builds use `goamd64: v3`.
- `checksums.txt` with SHA-256 hashes.
- `checksums.txt.sigstore.json` Cosign keyless signature bundle.

GitHub Release settings:

- `prerelease: auto`.
- Stable releases become latest; prereleases do not.
- `mode: replace`, so reruns may replace the GitHub Release content for the same tag.

## Supported User Installation Channels

- Shell install script for Linux/macOS downloads from GitHub Releases, verifies checksums, and installs to `~/.local/bin` by default.
- PowerShell install script for Windows downloads from GitHub Releases, verifies checksums, installs to `%LOCALAPPDATA%\Programs\invowk`, and updates User PATH.
- Homebrew cask `invowk/tap/invowk` is pushed by GoReleaser only for stable releases.
- WinGet package `Invowk.Invowk` is generated by GoReleaser only for stable releases; automation opens a PR to `microsoft/winget-pkgs`.
- `go install github.com/invowk/invowk@latest` follows module tags and is not a GoReleaser artifact.
- From-source installs follow the repository state and are not separately published.

## Stable Versus Prerelease

- Prereleases publish GitHub Release artifacts and checksums.
- Prereleases skip Homebrew and WinGet via `skip_upload: auto`.
- Stable releases publish GitHub Release artifacts, update Homebrew cask, create/enhance a WinGet PR, and trigger docs versioning.

## Secrets And Permissions

- `GITHUB_TOKEN`/GitHub App token: GitHub Release publishing. The workflow uses `actions/create-github-app-token` so release events can trigger docs workflows.
- `DOCS_APP_ID` and `DOCS_APP_PRIVATE_KEY`: required to generate the GitHub App token.
- `HOMEBREW_TAP_TOKEN`: fine-grained PAT with contents write on `invowk/homebrew-tap`.
- `WINGET_TOKEN`: classic PAT with `public_repo` for cross-repo WinGet fork PRs and enhancement.
- `BENCHER_API_TOKEN`: release benchmark tracking and Bencher registry image push.
- `id-token: write`: required for Cosign keyless signing. There is no Cosign private key.

## Post-Release Verification

```bash
tag=v1.2.3
gh release view "$tag" --json tagName,isPrerelease,assets,body
latest_tag=$(gh release view --json tagName --jq '.tagName')
test "$tag" = "$latest_tag" && echo latest || echo not-latest
gh run list --workflow Release --limit 5
gh run watch <run-id>
gh run list --workflow "Version Docs" --limit 5
```

For stable releases:

```bash
brew update
brew info --cask invowk/tap/invowk
gh pr list --repo microsoft/winget-pkgs --search "Invowk.Invowk $tag"
gh pr list --repo microsoft/winget-pkgs --head "invowk:invowk-${tag#v}"
```

Verify checksums and signatures from downloaded release assets:

```bash
cosign verify-blob \
  --bundle checksums.txt.sigstore.json \
  --certificate-identity-regexp='https://github\.com/invowk/invowk/.*' \
  --certificate-oidc-issuer='https://token.actions.githubusercontent.com' \
  checksums.txt
sha256sum -c checksums.txt
```
